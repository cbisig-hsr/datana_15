Video 2

Questions

  1. Can we always use a random sample for a validation set? Why or why not?
    Note: No, because the data in the validation set should come from an area where your predictions will find place e.g. Future
  2. What is overfitting? Provide an example.
    Note: When your model exactly learns to deal with your training data. (Fits perfectly to trainingdata but achieves bad results when using the model against data outside of the training set)
  3. What is a metric? How does it differ from “loss”?
    Note: A metric is a function that measures quality of the models predictions using the validation set. The loss function is used by the computer to decide how to update the parameters.
  4. How can pretrained models help?
    Note: By doing transfer learning -> fine tune a pretrained model to a specific problem
  5. What is the “head” of a model?
    Note: The part that has already been given in the pretrained model --> early layers
  6. What kinds of features do the early layers of a CNN find? How about the later layers?
    Note: Early: Diagonal lines, gradiants, specific colors -- Late: The combinations of features from previous layers - repeating patterns, and later more complex things like dogfaces etc.
  7. Are image models only useful for photos?
    Note: No they can be used for different problems as well - e.g. soud detection, fraud protection etc.
  8. What is an architecture?
    Note: The template of the model that we’re trying to fit.
  9. What is segmentation?
    Note: Splitting an object into different segments e.g Streets, cars, trees, road signals etc
  10. What is y_range used for? When do we need it?
    Note: The y_range is used to define the range of our target
  11. What are hyperparameters?
    Note: A hyperparameter is a parameter whose value is used to control the learning process
  12. What’s the best way to avoid failures when using AI in an organization?
    Note: Don’t use it for conversations
  13. What is a p value?
    Note: The probability of an observed result assuming that the null hypothesis is true.
  14. What is a prior?
    Note: Guess about what’s the most likely scenario
  15. Provide an example of where the bear classification model might work poorly in production, due to structural or style differences in the training data.
    Note: When there are many pictures that were resized in the trainig set but not in the actual input set
  16. Where do text models currently have a major deficiency?
    Note: Deep learning is currently not good at generating correct responses!
  17. What are possible negative societal implications of text generation models?
    Note: Spread disinformation, create unrest, and encourage conflic
  18. In situations where a model might make mistakes, and those mistakes could be harmful, what is a good alternative to automating a process?
    Note: Supervising the automated process or just partially automate the process
  19. What kind of tabular data is deep learning particularly good at?
    Note: Data with a big variety of columns.
  20. What’s a key downside of directly using a deep learning model for recommendation systems?
    Note: They only tell you what products a particular user might like, rather than what recommendations would be helpful for a user.
  21. What are the steps of the Drivetrain Approach?
    Note: 1) Define Objective 2) Decide what inputs can be controled (levers) 3) What data do we have and what data can we collect 4) Create the models
  22. How do the steps of the Drivetrain Approach map to a recommendation system?
    Note: 1) drive additional sales by surprising and delighting the customer with recommendations of items they would not have purchased without the recommendation 2) The lever is the ranking of the recommendations 3) New data must be collected to generate recommendations that will cause new sales 4) build two models for purchase probabilities, conditional on seeing or not seeing a recommendation
  23. Create an image recognition model using data you curate, and deploy it on the web.
    Warning: No solution in this blog post
  24. What is DataLoaders?
    Note: A fastai class that stores multiple DataLoader objects you pass to it, normally a train and a valid
  25. What four things do we need to tell fastai to create DataLoaders?
    Note: What kinds of data we are working with / How to get the list of items / How to label these items / How to create the validation set
  26. What does the splitter parameter to DataBlock do?
    Note: Tells the DataBlock how to split the data into the training and validation set
  27. How do we ensure a random split always gives the same validation set?
    Note: Set a fixed seed -> random numbers then will always be the same
